# 產品需求文件 (PRD) - 個人記帳簿

## 1. 專案概述

### 背景與動機
在現代忙碌的生活中，許多人對於自己的金錢流向不夠清楚，常常發生月底「吃土」的狀況。為了幫助人們建立良好的理財習慣，我們需要一個簡單易用、能快速記錄收支並提供直觀統計的個人記帳簿系統，讓使用者能清楚掌握自己的財務狀況。

### 目標用戶
- 有理財需求的大學生、上班族。
- 想要追蹤日常花費、控制預算的一般大眾。

### 核心價值主張
提供極簡、直覺的記帳體驗，結合清晰的圖表與數據比較，讓使用者一目了然地看到每個月的收支變化，進而達成財務管理的目標。

## 2. 功能需求

以下為系統的主要功能與對應的使用者故事：

### 功能一：記錄支出
- 作為 **一般用戶**，我希望 **能夠快速輸入花費金額與明細**，以便 **記錄我每天的支出情況**。

### 功能二：記錄收入
- 作為 **一般用戶**，我希望 **能夠輸入我的薪資或額外收入**，以便 **計算我目前的可用資金**。

### 功能三：分類支出與收入
- 作為 **一般用戶**，我希望 **能在記帳時選擇類別（如餐飲、交通、薪水等）**，以便 **知道我的錢都花在哪些地方**。

### 功能四：顯示餘額統計
- 作為 **一般用戶**，我希望 **在首頁能直接看到當前總餘額、當月總結算**，以便 **隨時掌握我還可以花多少錢**。

### 功能五：比較每個月的花費
- 作為 **一般用戶**，我希望 **能夠查看過去幾個月的收支圖表**，以便 **分析我的消費趨勢與檢討花費**。

## 3. 非功能需求

### 技術限制
- 前後端框架：使用 Flask 作為後端 Web 框架，結合 Jinja2 模板引擎渲染前端畫面。
- 資料庫：使用 SQLite 輕量級資料庫進行資料儲存。

### 效能與安全考量
- **效能**：確保系統操作反應迅速，頁面載入及資料儲存時間應在 2 秒內。
- **安全**：過濾使用者輸入內容，預防基本的 SQL Injection 及 XSS 攻擊。

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須有)
- 新增、編輯、刪除支出與收入紀錄。
- 提供基礎的收支分類選項。
- 顯示當月總收入、總支出與結餘。

### Should Have (應該有)
- 簡單的圓餅圖顯示各項支出的分類佔比。
- 長條圖比較最近幾個月的收支趨勢。

### Nice to Have (可以有)
- 支援使用者自訂收支分類。
- 資料匯出（如 CSV 格式）與備份功能。
- 支援預算設定與超支提醒。
